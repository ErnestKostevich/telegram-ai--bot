# Архитектура обновления AI DISCO BOT

## 1. Система BYOK (Bring Your Own Key)
Пользователи смогут выбирать провайдера и вставлять свой API ключ.

### Поддерживаемые провайдеры (10+):
1. **OpenAI** (GPT-4o, GPT-4o-mini)
2. **Anthropic** (Claude 3.5 Sonnet/Haiku)
3. **Google** (Gemini 1.5 Pro/Flash)
4. **Groq** (Llama 3, Mixtral) - супер быстрые
5. **OpenRouter** (Доступ ко всем моделям через один API)
6. **Mistral AI**
7. **Perplexity** (Для поиска в реальном времени)
8. **DeepSeek** (Дешевые и мощные модели)
9. **Together AI**
10. **Cohere**
11. **X.AI (Grok)**

### Структура данных (SQLAlchemy):
Добавляем таблицу `UserKeys`:
- `user_id` (BigInteger)
- `provider` (String)
- `api_key` (String, зашифрованный)
- `model` (String)
- `is_active` (Boolean)

## 2. Переработка системы чатов
Текущая проблема: жесткая привязка к Gemini и сложная логика контекста, которая может ломаться.

### Улучшения:
- **Абстрактный AI Клиент**: Единый интерфейс для всех провайдеров.
- **Улучшенное управление контекстом**: Хранение истории в БД (таблица `messages`), а не только в памяти.
- **Поддержка MarkdownV2**: Правильное форматирование ответов.
- **Streaming (по возможности)**: Для более быстрого визуального отклика.

## 3. План реализации
1. Создать `ai_providers.py` с классами для каждого провайдера.
2. Обновить `models.py` (вынести модели из `main.py`).
3. Создать меню `/settings` для выбора провайдера и ввода ключа.
4. Переписать `handle_message` для использования выбранного провайдера.
